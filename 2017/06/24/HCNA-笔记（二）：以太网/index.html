以太网卡

网络接口卡（Network Interface Card，NIC）通常也简称为“网卡”。网卡工作在数据链路层和物理层，同时具有数据链路层和物理层的功能。

  • 计算机上的网卡

从逻辑上说，网卡包含有7个功能模块：

  1. CU（Control Unit，控制单元）
  2. OB（Output Buffer，输出缓存）
  3. IB（Input Buffer，输入缓存）
  4. LC（Line Coder，线路编码器）
  5. LD（Line Decoder，线路解码器）
  6. TX（Transmitter，发射器）
  7. RX（Receiver，接收器）

  • 交换机上的网卡

计算机上的网卡是用来收发数据的，交换机上的网卡是用来转发数据的。
交换机上的网卡组成与计算机上的网卡的组成结构是完全一样的，除了 CU 外，各个功能模块的工作过程也是一样的。

计算机的网卡的 CU 需要进行帧的封装和解封，并与计算机上的 TCP/IP 的网络层交换数据；交换机的网卡的 CU 不需要进行帧的封装和解封，而是直接与交换机上其他网卡的 CU 进行帧的交换。

以太网帧

  • MAC地址

凡是符合 IEEE 802 标准的网络接口卡都必须拥有一个 MAC 地址。

每个 MAC 地址长为 48bit ，6个字节。

MAC 地址共有3种：

  1. 单播 MAC 地址：第一个字节的最低位是0，标识特定的网卡
  2. 组播 MAC 地址：第一个字节的最低位是1，标识一组网卡
  3. 广播 MAC 地址：每个比特都是1，标识所有的网卡

BIA 地址（Burned-In Address，固化地址，一种单播地址）的前三个字节是网卡制造商的 OUI（Organizationally-Unique Identifier），后三个字节是制造商自己确定的每块网卡独一无二的标识。

  • 以太帧格式

以太帧的格式有两种标准：一个是由 IEEE 802.3 定义的，称为 IEEE 802.3 格式；一个是由 DEC、Intel、Xerox 三家公司联合定义的，称为 Ethernet II 格式，也称 DIX 格式，是一种使用更加广泛的格式。

以下是关于Ethernet II 格式的以太帧的各个字段的描述。

  1. 目的 MAC 地址：有6个字节。可以是单播、组播、广播 MAC 地址。
  2. 源 MAC 地址：有6个字段。只能是单播地址
  3. 类型：有2个字节，表示载荷的类型。
    • 0x0800 ：IPv4 数据包
    • 0x86dd ：IPv6 数据包
    • 0x0806 ：ARP数据包
    • 0x8848 ：MPLS 报文
  4. 载荷数据：长度可变，46~1500字节。是该帧的有效载荷。
  5. CRC 字段：有 4 个字节。对帧进行检错校验。

根据目的 MAC 地址的不同，可以将帧分为3种类型：

  • 单播以太帧（单播帧）：目的 MAC 地址为一个单播 MAC 地址的帧
  • 组播以太帧（组播帧）：目的 MAC 地址为一个组播 MAC 地址的帧
  • 广播以太帧（广播帧）：目的 MAC 地址为一个广播 MAC 地址的帧

以太网交换机

交换机转发数据的端口都是以太网口，这样的交换机称为以太网交换机。

  • 3 种转发操作

交换机的基本作用就是用来转发数据帧的。交换机对于传入的帧一共有3种转发操作：

  • 泛洪（Flooding）：交换机把从某一端口接收到的帧通过其他所有端口转发出去（不包括数据帧进入交换机的端口）
  • 转发（Forwarding）：交换机把从某一端口接收到的帧通过另一端口转发出去
  • 丢弃（Discarding）：交换机把从某一端口接收到的帧直接丢弃

  • MAC 地址表

    MAC 地址表的老化时间默认为 300s。

    在计算机中的 MAC 地址表没有老化时间。

  • 交换机工作原理

概括如下：

  • 如果传入交换机的是一个单播帧，交换机会在 MAC 地址表中查找这个 MAC 地址。
    1. 如果查不到这个地址，执行泛洪操作
    2. 如果查到了这个地址，则比较这个 MAC 地址在地址表中对应的端口编号是不是这个帧进入交换机的那个端口的编号.
      2.1 如果不是，执行转发操作
      2.2 如果是，执行丢弃操作
  • 如果传入的是广播帧，交换机不会去查 MAC 地址表，直接执行泛洪操作
  • 如果传入的是组播帧，略。

ARP Address Resolution Protocol

ARP 协议的基本作用是根据 IP 地址获得对应的 MAC 地址，是一个网络层协议。

  • ARP 协议基本原理

如上图所示，假设源主机 PC1 已经知道了目的主机 PC2 的 IP 地址为 10.0.0.2，现状需要获取 PC2 的 MAC 地址。过程如下。

  1. PC1 发送一个广播帧，该广播帧的源 MAC 地址为 MAC1，类型字段为 0x0806，表示是一个 ARP （请求）报文。该 ARP 报文的意思是：我的 IP 地址是 10.0.0.1，MAC 地址是 MAC1，请问 IP 地址为 10.0.0.2 所对应的 MAC 地址是多少？
  2. 因为 PC1 发送的是广播帧，所以 PC2 和 PC3 都会接收到它。并交由上层的 ARP 处理模块处理
  3. PC3 的 ARP 处理模块 会发现请求中的 10.0.0.2 并不是自己的 IP 地址，所以不会应答，而是将报文中的10.0.0.1 与 MAC1 的对应关系存放在自己的 ARP 缓存表中，然后丢弃报文
  4. PC2 的处理模块发现 10.0.0.2 是自己的 IP 地址，所以会进行应答。PC2 会发送一个单播帧到 PC1 ，该帧的目的 MAC 地址为 MAC1 ，源 MAC 地址为 MAC2，类型值还是 0x0806。该帧实际上是一个应答报文，报文中还包含由 PC2 的 IP 地址与 MAC 地址。另外，PC2 也会将请求报文中的 10.0.0.1 与 MAC1 的对应关系存放到自己的 MAC 缓存表中。
  5. PC1 收到 PC2 发送的单播帧后，会将数据交由上层 ARP 处理模块进行处理。处理模块会提取出 PC2 的 IP 地址与 MAC 地址的对应关系，并存放到自己的 MAC 缓存表中。

上边提到了 MAC 缓存表的概念，MAC 缓存表是设备用来临时存放 IP 地址与 MAC 地址的对应关系的。当某一设备想要发送单播帧时会首先查看自己的 MAC 缓存表中是否存在目的设备的 MAC 地址，若有，则直接发送帧；没有，则要发起 ARP 请求来获得 MAC 地址。

MAC 缓存表中每个条目的默认生存期时 180s。过了生存期，该条目就会被删除。

  • ARP 报文格式

ARP 报文分为请求报文和应答报文，两者的结构是相同的，只是其中某些字段的取值不同。

字段 含义
硬件类型 长度2个字节。表示网络类型，以太网取值为1.
协议类型 长度2个字节。表示协议地址类型；取值为 0x0800 即表示根据 IP 地址来进行映射
硬件地址长度（hln） 长度1个字节。表示硬件地址的长度，以太网中取值为6，表示 MAC 地址长度为6个字节
协议地址长度（pln） 长度1个字节。表示协议地址的长度；取值为4表示 IP 地址的长度为4个字节
op 长度2个字节。表示 ARP 报文的类型。取值为1表示 ARP 请求报文；取值为2，表示 ARP 应答报文
PAD 一共有18个字节，目的是为了凑足以太网的载荷数据最小为48字节
